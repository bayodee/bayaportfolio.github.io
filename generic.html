24/7 online banking system for
baz finance

In this project, I introduced an online banking system which enables customers log in to and manage their accounts 24 hours of the day, thereby,
significantly reducing call center volume, and expenses as a whole.

Banking Business Requirements Document

Project View

For BAZ Finance

1. The Business Problem

Currently, Baz Finance offers limited banking on weekdays (8 hours per day). This does not provide enough service hours for the customers as the bank is not open in the evenings or on weekends.

Moreover, when a customer calls the contact center, it takes them at least 20 minutes before they can speak to an agent. This results in a bad customer service experience.

Additionally,phone calls made by customers costs about $10 per hour, which represents a significant business expense.

2. The Proposed Solution

Baz Finance would like to introduce Online Banking capabilities. This feature will allow customers to login to their accounts and manage them 24x7.

This would also significantly reduce call center volume and help lower the expenses associated with running a call center.

3. System(s) Impacted

Online Banking capabilities will be added to the customer-facing website: bazfinance.com.

5. Business Requirements


A. Core Features/ Functional Requirements

User Authentication and Authorization

Requirement: Secure user registration, login, and multi-factor authentication.

  • Implement OAuth2 and support biometric login (fingerprint, Face ID).
  • Develop user onboarding, including KYC verification.
  • Integrate multi-factor authentication for transactions and login.

Digital Wallet

Requirement: Enable users to store, deposit, and withdraw funds.

  • Develop wallet management system (balance tracking, transaction history).
  • Implement fund deposit methods (bank transfer, card payment, QR code).
  • Implement fund withdrawal methods (bank transfer, cash out at agents).
  • Ensure reconciliation with bank APIs.

Payments and Transfers

Requirement: Support peer-to-peer transfers, bill payments, and merchant payments.

  • Enable in-app payments (transfers to other app users).
  • Integrate with payment gateways for utility bills and airtime.
  • Implement merchant payments via QR code or unique merchant ID.
  • Develop real-time transaction alerts and notifications.

Banking Features

Requirement: Provide bank accounts, debit cards, and virtual cards.

  • Build bank account creation and management.
  • Integrate with card issuance services for debit/virtual cards.
  • Implement card controls (e.g., set limits, freeze card, etc.).

Lending/Investment Services (Optional)

Requirement: Allow users to access small loans and investment options.

  • Develop loan application, processing, and disbursement.
  • Implement investment options (e.g., fixed deposits, savings plans).
  • Add loan repayment scheduling and reminders.

Agent Network (Optional)

Requirement: Establish a network of agents for cash-in/cash-out.

  • Design and implement agent registration and management.
  • Develop agent commission tracking.
  • Implement agent location mapping.

B. Non-Functional Requirements

Scalability

Requirement: Ensure app can handle a large volume of users and transactions.

  • Develop scalable backend infrastructure (use microservices if needed).
  • Implement caching for high-frequency data.
  • Load test the system to determine maximum capacity.

Security

Requirement: Safeguard user data and transactions.

  • Implement encryption protocols.
  • Set up fraud detection and alerts.
  • Conduct vulnerability assessments and regular security audits.

Performance

Requirement: Optimize speed and responsiveness.

  • Optimize database queries and implement indexing.
  • Conduct performance testing to minimize latency.
  • Add efficient data storage solutions for high-frequency data.

C. User Experience (UX) Requirements

User Interface

Requirement: Ensure clean, user-friendly, and accessible UI.

  • Design user-friendly layouts and workflows.
  • Develop UI/UX for key flows (login, payments, transfers, etc.).
  • Conduct A/B testing to refine features.

Localization and Language Support

Requirement: Support multiple languages if targeting various regions.

  • Implement multi-language support.
  • Conduct localization testing for UI/UX elements.

Accessibility

Requirement: Ensure app is accessible to all users, including those with disabilities.

  • Add support for screen readers and high-contrast modes.
  • Develop a simplified layout for accessibility modes.

D. Compliance and Integration Requirements

Data Privacy

Requirement: Ensure compliance with data privacy laws.

  • Implement user consent features and privacy settings.
  • Ensure user data is securely stored, anonymized, or encrypted.
  • Develop user data export and deletion functionalities.

Financial Regulations

Requirement: Comply with financial regulatory requirements.

  • Set up AML monitoring and reporting.
  • Implement transaction limits and flagging for suspicious activity.
